Redux in React

Posted on 2020-12-06 In Javascript

Redux is not Redundant.

Concepts of Redux#

The three most important parts in Redux are Store, Reducer and Actions.

The Store is a central data warehouse for storing data in the app as the app’s state, being the only authorized data sources for data in the app. The Actions can have many Types (usually in contant string format), the behavior of each Type of Action is predefined, the events in the React app can trigger certain Actions, and those Actions will be dispatched to the Store. Then the Reducers will update the state in the Store according to the Actions which are dispatched to the Store, notice that they should only update the states in the Store (no other tasks), and then the Store will notify the React app there is a newer state and pass a newer state to the app.

Dependencies in React#

Installing packages redux, react-redux, redux-thunk using npm install --save ...

Enable Redux in React app#

  1. Define the types of actions will be conducted and the Action creator
  2. Define the Reduces for handling those Actions
  3. Create a Store instance, pass the rootReducer as the argument to the store instance
  4. Wrap the whole app component with tag <Provider store={store}> </Provider>
  5. In the component, use connect to connect the component to the global store
  6. One can also mapStateToProps for each component, then the state in store can be mapped into component’s state
